# Site Search (Local)

Nextra 4 uses [Pagefind](https://pagefind.app) for fully client-side, zero-backend local search. No external search service, API key, or hosted index is required: Pagefind crawls the statically rendered HTML output and produces a static search index that ships alongside the site.

That index is generated by this project's `postbuild` script, `pagefind --site .next/server/app --output-path public/_pagefind`, which runs automatically after `next build`. Because the index is built from the compiled output rather than the dev server, search only works against a production build — running `bun run build && bun run start` (or the npm equivalents) serves a working search box, while `next dev` shows no results since no index has been generated yet.

The search box lives in the navbar and is reachable from any page, with a keyboard shortcut (typically Cmd/Ctrl+K) for quick access.
