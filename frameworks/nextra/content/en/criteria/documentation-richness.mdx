# Documentation Richness

Nextra's own official documentation at https://nextra.site is thorough: it covers guide and how-to content, a full theme configuration reference for both the Docs theme and the Blog theme, file conventions (such as `_meta.ts`, `page.tsx`, and content directory layout), and reference material for its built-in components. The official site is itself built with Nextra, which is a form of dogfooding — the maintainers rely on the same rendering pipeline, search, and theme that ship to users.

Documentation richness and quality are inherently more subjective than the other, more mechanically-verifiable criteria in this comparison (such as whether search works, or whether a route 404s). Readers should weigh this note as one data point alongside their own reading of the official docs rather than as a substitute for it.
