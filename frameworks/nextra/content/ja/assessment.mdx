# Nextra の評価

このページでは、**ハンズオンでの印象（主観）**と、このリポジトリおよび公開履歴で
確認した**根拠**を分けて記載します。

## 結論

> **ハンズオンでの印象（主観）:** Nextra のドキュメントテーマ、UX、配色は洗練されて
> おり、静的エクスポート時の画像オーサリングとレイアウトの挙動も良好に感じられました。
> Next.js と React を採用済みで、統合とエクスポートのトレードオフを受け入れられる
> チームに有力な選択肢です。

## 強み

- **ハンズオンでの印象（主観）:** デフォルトの Docs テーマは洗練されており、
  カスタマイズの余地もあります。
- **根拠:** このサンプルは事前レンダリング済みの静的HTMLを出力し、多言語コンテンツ、
  Pagefind によるローカル検索、ダークモード、MDXコンポーネントを備えています。
- **根拠:** 静的 import と `staticImage` は、Next.js の画像オプティマイザを使わない
  このデプロイでも、寸法情報と安定した遅延読み込み・レイアウト挙動を保ちます。

## 懸念点とトレードオフ

- **ハンズオンでの印象（主観）:** 今回は Nextra のクイックスタートが VitePress より
  明確に手数が多く感じられました。Next.js、React、Nextra、テーマに加え、設定と
  App Router の layout/catch-all/MDX の配線が必要です。この二言語静的 Pages
  サンプルではさらに、辞書、リダイレクトの回避策、Pagefind、export 設定が必要です。
  これはこのビルドでの所見であり、すべての Nextra プロジェクトが難しいという意味では
  ありません。
- **ハンズオンでの印象（主観）:** 公式ドキュメントは Docs/Blog の基本セットアップと
  APIページを扱っていますが、今回の評価では、深いカスタマイズや移行の詳細は
  examples・ソース・Issue履歴も必要になる程度に少ないと感じました。
  [Blog Theme の開始ガイド](https://nextra.site/docs/blog-theme/start) と
  [そのドキュメントツリー](https://github.com/shuding/nextra/tree/main/docs/app/docs/blog-theme)
  から確認できますが、Blog Theme に完全なリファレンスがあることを意味するものでは
  ありません。
- **根拠:** 本番では `images: { unoptimized: true }` を使います。静的画像の
  メタデータはオーサリングとレイアウトに役立ちますが、この GitHub Pages export は
  リサイズ、形式変換、オプティマイザへのリクエストを行いません。

## VitePress との共通点

**共通するハンズオンでの印象（主観）:** どちらのデフォルトテーマも洗練されていると
感じられ、カスタマイズ手段も用意されています。

**両サンプルで確認した根拠:** いずれも事前レンダリング済みHTMLを出力してから
hydration します。基本テキストはHTMLに含まれ、拡張された検索、ナビゲーション、
コンポーネントは JavaScript を使います。英語/日本語の対応は本プロジェクトが執筆した
もので、自動翻訳ではありません。エコシステムのトレードオフも異なります。Nextra には
Next/React と静的エクスポートの制約があり、VitePress ではサードパーティプラグインに
依存する場面があります。

## 選定の指針

Next.js/React との整合性、Docs 体験、MDX統合が、追加の設定と静的エクスポートの制約を
上回るなら Nextra を選ぶ価値があります。フレームワークの配線を最小化することを
優先するなら、より軽量な選択肢を検討してください。深いカスタマイズや移行を決める前に、
必要なドキュメントを確認することを勧めます。この出力は、SPA風のクライアント遷移を
備えた複数ルートの静的出力です。JavaScript なしの MPA モードは文書上見つかりません
でしたが、hydration 自体がSEOで不利というわけではありません。
