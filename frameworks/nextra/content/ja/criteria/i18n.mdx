# 多言語対応（i18n）

Nextra の App Router 向け i18n は、`content/<locale>/` ディレクトリ規約・`[lang]`
動的ルートセグメント・`nextra/locales` から `proxy` を re-export する `proxy.ts`
に基づいた、ロケール接頭辞付きルート（例: `/en`, `/ja`）を使用する。`proxy.ts`
はリクエストを検査し、接頭辞なしのパスにアクセスした訪問者を優先言語へ自動的に
リダイレクトする。ナビバーに表示される `<LocaleSwitch>` コンポーネントを使えば、
この自動判定とは独立していつでも手動でロケールを切り替えられる。

このサイト自体が、完全な多言語パリティを実現している実例である。英語に存在
するページは全て日本語にも存在し、その逆も同様である。パリティが欠けていると、
訪問者が言語切替を行った際にリンク切れや404に遭遇してしまうためだ。実際、この
問題は本プロジェクトの開発中に見つかり修正されたものであり、それ以降ロケール
ごとのコンテンツ網羅性は、前提とせず明示的に確認すべき対象として扱われている。
本サイトの既定言語（`defaultLocale`）は英語（`en`）であり、`next.config.mjs`
で設定されている。
