# サイト内検索（ローカル）

Nextra 4 は、完全にクライアントサイドで動作しバックエンドを必要としないローカル
検索エンジン [Pagefind](https://pagefind.app) を使用する。外部の検索サービス・
APIキー・ホスト型インデックスは不要で、Pagefind は静的にレンダリングされた
HTML出力をクロールし、サイトと一緒に配信される静的な検索インデックスを生成する。

そのインデックスは、`next build` の後に自動実行される本プロジェクトの
`postbuild` スクリプト（`pagefind --site .next/server/app --output-path
public/_pagefind`）によって生成される。インデックスはビルド済みの出力から
作られるため、検索は本番ビルドに対してのみ機能する。`bun run build && bun run
start`（あるいは npm 相当のコマンド）を実行すると検索ボックスが動作するが、
`next dev` の開発サーバではまだインデックスが生成されていないため結果が
表示されない。

検索ボックスはナビバーにあり、どのページからもアクセスできる。キーボード
ショートカット（通常 Cmd/Ctrl+K）でも素早く開ける。
