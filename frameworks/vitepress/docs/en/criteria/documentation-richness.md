# Documentation Richness

VitePress has an extensive, maintained official documentation site that covers
the product from installation through theme APIs. Particularly relevant
starting points for this comparison are:

- [Getting started](https://vitepress.dev/guide/getting-started) for project
  setup and commands.
- [Markdown extensions](https://vitepress.dev/guide/markdown) for alerts,
  containers, code blocks, and math configuration.
- [Asset handling](https://vitepress.dev/guide/asset-handling),
  [i18n](https://vitepress.dev/guide/i18n), and
  [routing](https://vitepress.dev/guide/routing) for site structure.
- [Default-theme configuration](https://vitepress.dev/reference/default-theme-config)
  and [API reference](https://vitepress.dev/reference/site-config) for exact
  configuration fields.

The documentation also includes deploy guidance and framework examples. As
with any framework evaluation, “richness” is qualitative: the links above are
evidence of breadth, while the pages in this site verify the specific behavior
used by this project.

VitePress's current v2 prerelease transition adds a version-selection step:
read the documentation for the installed version and confirm significant
configuration changes in a build. The dated [assessment](/en/assessment)
records the release-cadence context rather than treating broad documentation
as a guarantee of cross-version compatibility.
