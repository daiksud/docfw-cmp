# 国際化

**対応状況: 標準。** VitePress の `locales` 設定により、日本語は `/ja/` で配信され、HTML 言語は `ja-JP` になります。組み込みの言語切替は現在の相対パスを保ち、たとえば `/ja/criteria/gfm/` から `/en/criteria/gfm/` へ移動します。

| 対象 | 日本語側の扱い |
| --- | --- |
| コンテンツ | `docs/ja/` に置く |
| ルート | `/ja/...` |
| サイト名と説明 | 日本語ロケールで設定 |
| 検索 UI | ロケールごとの翻訳を設定 |

::: info 翻訳は自動ではない
ロケール設定はルーティングと UI を切り替えますが、Markdown 本文や対応ページを自動生成しません。このプロジェクトでは `docs/en/` と `docs/ja/` のページ構成を一致させ、同じ項目へ安全に切り替えられるようにしています。
:::

[日本語のトップページ](/ja/) に戻ると、このロケールのサイドバーとナビゲーションを確認できます。
